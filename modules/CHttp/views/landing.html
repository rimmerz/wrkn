WorkOn (component based application)

Main things:

  1. Auth component (API/WEB) affects (routes, access)
  2. RBAC component (WEB) affects (routes, access, another components)
  3. Cascade Permissions Component (WEB) affects (any part of application based on components)
  4. Dynamic Tree based Component module (WEB) affects (views, partials, rendering parts, any application part)
    1. Depends from C.P.C.
    2. Base View component
    3. Partial View component
    4. Rendering parts component
  5. TableDataProvider Component (WEB) (Based on php CRM TableDataProvider ideology)
    1. Depends from C.P.C.
    2. Backend layer
    3. Frontend layer
  6. Notification Feed Component (WEB) (Based on php CRM TableDataProvider and CSocketTransport ideology)
    1. Backend layer
    2. Frontend layer
  7. Global Search Map Component (WEB)
    1. Depends from C.P.C. and related to D.T.C.
    2. Backend layer
    3. Frontend layer
  8. Translation Transfer Component (WEB)
    1. Backend layer
    2. Frontend layer
  9. Form Delivery Component (WEB)
    1. Frontend layer
  10. Document Generator Component (API/WEB)
    1. Backend layer (Partially depends from F.D.C.) depends from C.P.C and related to D.T.C.

Core things:

  1. Behaviours (Any step of lifecycle based on before{SomeMethod} after{SomeMethod})
  2. Scenarios (Any data processing schema step by step)
  3. Events (Placing event triggers and listeners to any part of code with registration in nosql db for ability enabling/disabling and monitoring)
